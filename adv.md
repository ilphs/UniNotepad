# UniNotepad 추가 기능 우선순위 (Advanced Roadmap)

> "가볍게 사용할 수 있다"는 전제 하에 전체 검토 후 정리한 기능 추가 우선순위.
> 작성일: 2026-07-13

## 현재 기능 (baseline)

탭(생성/닫기/드래그 재배열/Cmd+1~9 전환) · 세션 자동 복원 · 17개 언어 구문강조 · 인코딩/EOL 자동감지 · 줌 · Find/Replace · 라이트/다크/시스템 테마 · 상태바(줄·열·선택·줌·EOL·인코딩) · 파일 드래그&드롭 · 외부 변경 충돌 감지 · 자동 줄바꿈

---

## P0 — 이미 있는 걸 "완성"하는 저비용·고가치 (가장 먼저)

| 기능 | 왜 | 비용 | 관련 파일 |
|------|-----|------|-----------|
| **상태바 인코딩/EOL 클릭 변경** | 지금은 `statusbar.ts`에서 **표시만** 하고 바꿀 수 없음. Notepad++ 핵심 UX. 클릭 → 드롭다운 → 재저장(`save_file`는 이미 인코딩/EOL 파라미터 받음) | 낮음 | `statusbar.ts`, `tabs.ts` |
| **Go to Line (Cmd+G)** | 로그·설정파일 편집 시 필수. CM6에 줄 이동 명령 연결 + 메뉴/단축키 | 낮음 | `editor.ts`, `menu.rs`, `menu.ts` |
| **자동 괄호쌍/괄호매칭/입력 들여쓰기** | `editor.ts` 확장 목록에 `closeBrackets()`, `bracketMatching()`, `indentOnInput()`가 빠져 있음. 코드 편집 체감 크게 향상, CM6 기본 제공이라 거의 공짜 | 낮음 | `editor.ts` |
| **Tab 들여쓰기 키맵** | 현재 `indentWithTab` 미등록이라 Tab 키가 들여쓰기로 동작하지 않음 | 매우 낮음 | `editor.ts` |

## P1 — 가벼움 유지하며 체감 큰 것

| 기능 | 왜 | 비용 |
|------|-----|------|
| **자동 줄바꿈 On/Off 토글** | 지금은 `lineWrapping` 항상 켜짐. 로그/CSV엔 끄고 싶은 수요. View 메뉴 토글 + 저장 | 낮음 |
| **닫은 탭 다시 열기 (Cmd+Shift+T)** | 실수로 닫은 탭 복구. 세션 백업 인프라가 이미 있어 재활용 가능 | 중간 |
| **최근 파일 목록 (File > Recent)** | 가벼운 에디터의 표준 편의. 경로 리스트만 저장 | 중간 |
| **글자수/단어수 상태바** | `statusbar.ts`에 한 줄 추가 수준 | 낮음 |
| **저장 시 옵션**(끝 공백 제거 / 마지막 개행 보장) | 협업/린트 친화적. 설정 토글 | 낮음 |

## P2 — 선택적 (있으면 좋지만 없어도 됨)

- **폰트 패밀리/크기 설정 UI** — 현재 줌만 있고 폰트는 CSS 고정
- **공백/제어문자 표시(show invisibles)** — CM6 확장
- **코드 폴딩(foldGutter)** — 긴 파일용, 다만 "가벼움"과는 약간 상충
- **탭 많을 때 오버플로 드롭다운** — 지금은 가로 스크롤만
- **간단한 자동완성** — 무게 증가 주의

---

## 가볍게 유지하려면 "피하거나 미루기"

스코프가 커지고 무거워지는 것들 — 명시적 요구 전엔 넣지 않길 권장:
- 다중 파일 검색(Find in Files) / 프로젝트 트리 · 사이드바
- 플러그인/확장 시스템 · 커맨드 팔레트
- LSP·린터 통합 · Git 연동 · 미니맵 · 분할(split) 뷰

## 추천 실행 순서

- **1차(반나절급):** P0 4개 — 특히 **상태바 인코딩/EOL 변경**과 **괄호쌍/들여쓰기 확장**이 "미완성 느낌"을 가장 크게 없앰.
- **2차:** 자동 줄바꿈 토글 → 닫은 탭 복구 → 최근 파일.
